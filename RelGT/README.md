# How to run experiments using the RelGT architecture

## Changes made to the original RelGT code

### 1. On-the-fly subgraph sampling with NeighborLoader

The original RelGT code precomputed and stored subgraphs for all seed nodes before training. We replaced this with on-the-fly subgraph sampling using PyG's `NeighborLoader`, so that the sampling behavior is directly controlled by a per-edge-type fanout config — either a MetaSieve `.json` config or an MPS-GNN `.py` config.

### 2. Adapting NeighborLoader batches to the RelGT token pipeline

On-the-fly sampled subgraphs are adapted to the RelGT token-based input format using the `relgt_from_neighborloader_batch()` function in `utilsv2.py`.

### 3. Modified files

`main_node_ddp.py` and `utils.py` are the original RelGT code. To support on-the-fly subgraph sampling with `NeighborLoader`, we wrote `main_node_ddp_v2.py` and `utilsv2.py` with the necessary modifications. All experiments should be run using `main_node_ddp_v2.py`.

---

## Pruning configs

The `pruning_configs/` directory contains pre-generated sampling configs for all datasets and tasks used in the paper, organized as:

```
pruning_configs/
└── <dataset>/
    └── <task>/
        ├── <task_prefix>_<N>hops.py      # MPS-GNN sampling config
        └── <task_prefix>_pruned_<N>hops_<threshold>.json  # MetaSieve sampling config
```

- `.py` configs were generated by the MPS-GNN pipeline (see `../MPS-GNN`).
- `.json` configs were generated by the MetaSieve pruning pipeline (see `../MetaSieve`).

## Running experiments

All experiments are launched from `expts/run-single.sh`. The key variable is `--nl_num_neighbors`, which controls the sampling strategy.

A second important parameter is `--num_neighbors`, which sets **K** — the number of neighbor tokens each seed node attends to in the RelGT transformer. This is independent of the subgraph sampling fanout: the subgraph is sampled first, and then K neighbors are selected from it to form the token sequence. Refer to our paper for the value of `--num_neighbors` / K used for each dataset and task.

### a) Random sampling

Pass a comma-separated fanout string (one value per hop):

```bash
--nl_num_neighbors "64,64,64"
```

### b) MetaSieve sampling

Pass the path to a `.json` pruning config prefixed with `@`:

```bash
--nl_num_neighbors "@pruning_configs/rel-stack/user-badge/user_badge_pruned_3hops_0.1.json"
```

### c) MPS-GNN sampling

Pass the path to a `.py` sampling config prefixed with `@`:

```bash
--nl_num_neighbors "@pruning_configs/rel-stack/user-badge/user_badge_3hops.py"
```

---

### Example: `expts/run-single.sh`

The script contains three blocks — one for each sampling mode — with two commented. Edit the `RUN_NAME`, `--out_dir`, and `--nl_num_neighbors` to switch between modes. Other hyperparameters (`--batch_size`, `--epochs`, `--lr`, etc.) stay the same across all three modes.

Run from the repo root:

```bash
cd expts/
bash run-single.sh
```
